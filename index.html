    EspCure

    Open-source DIY cannabis curing chamber controller.
    Cannatrol-class dew point control for the price of a wine fridge.

    What it does

    🌡️

    Heat-Only PID Temperature

    The heater chases temperature with a tunable PID (default 60 °F / 15.6 °C); the Peltier chases dew point. A high-temp safety ceiling forces cooling if the chamber ever overheats. Built-in autotune button.

    💧

    Dew Point + VPD

    Controls humidity by dew point — exactly how the Cannatrol works — by running the Peltier cold plate to condense moisture. Switch between dew-point and VPD modes in Home Assistant. VPD shown as a live sensor.

    🌱

    Auto Cure Program

    Enable the 10-Day Dry recipe and walk away. It ramps dew point gently then holds at precise setpoints, stepping automatically at midnight. No manual control needed.

    🏠

    Home Assistant Native

    Encrypted native API integration. Climate entity, all sensors, setpoint numbers, and program switches show up instantly. OTA updates, fallback AP, local web UI.

    🛡️

    Frost Protection

    Software frost floor keeps the chamber from over-cooling. Configurable minimum temperature — PID suspends automatically and resumes when the chamber recovers.

    🔧

    One config file

    Everything lives in espcure.yaml. Validate with esphome config, flash via USB or OTA. No cloud dependency, no subscription, no app.

    Core hardware

    Controller ESP32-C6 DevKit
    Sensor SHT45 (I²C)
    Cooling Honeywell thermoelectric fridge + 2× TEC
    Heating 12 V 50 W PTC + fan
    Switching 3× SSR-40 DD (fans / TECs / heater)
    Power 12 V 300 W PSU

    Quick start

    1
    Clone & configure 
    git clone https://github.com/kinggh0stee/Espcure
cp secrets.yaml.example secrets.yaml
# fill in WiFi + API key
    2
    Validate 
    esphome config espcure.yaml
    3
    Flash (USB first time) 
    esphome run espcure.yaml
    4
    Add to Home Assistant

    Settings → Devices & Services → ESPHome → espcure.local
